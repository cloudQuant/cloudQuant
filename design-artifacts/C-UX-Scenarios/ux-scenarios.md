# UX 场景 (UX Scenarios)

## 概述

本文档描述 cloudQuant 平台的核心用户体验场景，包括用户画像、用户旅程和交互设计。

### 设计原则

1. **简洁优先**: 降低学习曲线
2. **数据驱动**: 清晰展示所有数据
3. **安全第一**: 风险始终可见
4. **即时反馈**: 每个操作都有响应

## 核心场景

### 场景 1: 策略开发与回测

#### 触发条件

- 用户点击「创建新策略」
- 或选择「导入策略模板」

#### 前置条件

- 用户已登录
- 用户有基本策略概念

#### 主流程

```text
步骤 1: 选择策略类型
├── 选择市场 (加密货币/股票/期货)
├── 选择策略类型 (趋势/均值回归/套利)
└── 选择时间框架 (日内/波段/长期)

步骤 2: 编写/生成策略代码
├── AI 辅助生成
│   ├── 输入策略描述
│   ├── AI 生成代码
│   └── 用户调整优化
├── 手动编写
│   ├── 使用代码编辑器
│   ├── 语法高亮
│   └── 自动补全
└── 导入模板
    ├── 浏览策略库
    ├── 预览策略逻辑
    └── 选择并导入

步骤 3: 配置回测参数
├── 选择交易品种
├── 设置时间范围
├── 配置初始资金
├── 设置手续费率
└── 选择滑点模型

步骤 4: 运行回测
├── 显示进度条
├── 实时更新统计
└── 预估剩余时间

步骤 5: 查看回测结果
├── 绩效概览
│   ├── 总收益率
│   ├── 夏普比率
│   ├── 最大回撤
│   └── 胜率
├── 收益曲线图
├── 交易记录列表
├── 风险分析报告
└── 优化建议

步骤 6: 保存/分享
├── 保存到我的策略
├── 导出报告
└── 分享到社区
```

#### 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 代码语法错误 | 高亮错误行，提供修复建议 |
| 数据缺失 | 提示缺失范围，建议替代方案 |
| 回测超时 | 显示部分结果，提供继续选项 |
| 内存不足 | 建议缩小范围或分批处理 |

#### 后置条件

- 策略已保存
- 回测结果可查看
- 可进行实盘部署

│   ├── 当前持仓

├── 价格预警

#### 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 网络断开 | 自动重连 + 通知用户 |
| API 限流 | 自动降速 + 预警 |

    ├── 显示建议
    ├── 一键执行
    └── 自定义调整

## 边缘场景

### 场景 E1: 网络中断处理

```text
检测:
- WebSocket 心跳超时
- API 请求失败

处理流程:
1. 显示断开提示
2. 尝试自动重连 (最多3次)
3. 重连成功: 恢复正常，同步数据
4. 重连失败: 
   - 保存当前状态
   - 建议检查网络
   - 提供离线模式 (查看历史数据)

恢复:
- 重新连接后同步所有数据
- 补充缺失期间的关键事件
```

### 场景 E2: 极端市场行情

```text
检测:
- 价格波动超过阈值
- 交易量异常
- 流动性枯竭

处理流程:
1. 触发高风险预警
2. 建议暂停策略
3. 自动执行保护性措施:
   - 取消待成交订单
   - 限制新开仓
   - 触发止损订单
4. 记录异常事件
5. 生成事后分析报告
```

### 场景 E3: 系统过载

```text
检测:
- CPU/内存使用率过高
- 请求延迟增加
- 队列堆积

处理流程:
1. 触发降级模式
2. 优先保证核心功能:
   - 订单执行
   - 风险监控
   - 止损触发
3. 暂停非核心功能:
   - 回测
   - 报告生成
   - 历史数据查询
4. 通知用户
5. 自动扩容 (如启用)
```
