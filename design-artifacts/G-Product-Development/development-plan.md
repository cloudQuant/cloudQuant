## 5. 代码管理

### 5.1 分支策略

```text
main (生产)
  │
  ├── develop (开发)
  │     │
  │     ├── feature/xxx (功能)
  │     ├── bugfix/xxx (修复)
  │     └── refactor/xxx (重构)
  │
  └── release/x.x (发布)
        │
        └── hotfix/xxx (紧急修复)
```

### 5.2 提交规范

```text
<type>(<scope>): <subject>

类型:
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

示例:
feat(backtest): 添加多品种回测支持
fix(order): 修复订单重复问题
docs(api): 更新 API 文档
```

### 5.3 代码审查

**审查清单**:

- [ ] 代码风格符合规范
- [ ] 有足够的测试覆盖
- [ ] 文档已更新
- [ ] 无安全漏洞
- [ ] 性能可接受

