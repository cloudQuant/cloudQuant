# 工作流创建指南

本指南介绍如何创建自定义 BMAD 工作流。

## 📋 概述

工作流是 BMAD 框架中定义任务执行流程的核心概念。一个工作流：

- 定义了任务的执行顺序
- 指定了每个步骤使用的代理
- 描述了输入和输出的数据流

## 🏗 工作流结构

```markdown
# 工作流名称

## 元数据

- 触发条件
- 前置条件
- 预期输出

## 步骤

1. 步骤 1
2. 步骤 2
   ...

## 异常处理

- 错误场景
- 恢复策略
```

## 🚀 创建工作流

### 步骤 1: 确定工作流目标

明确回答以下问题：

- 这个工作流解决什么问题？
- 谁会使用这个工作流？
- 预期的输出是什么？

### 步骤 2: 定义触发条件

```markdown
## 触发条件

### 自动触发

- 当用户请求 X 时
- 当文件 Y 被修改时

### 手动触发

- 用户显式调用工作流
```

### 步骤 3: 设计工作流步骤

```markdown
## 步骤

### 步骤 1: 数据收集

- **代理**: data-collector
- **输入**:
  - source: 数据源配置
- **输出**:
  - data: 收集的原始数据
- **超时**: 5分钟

### 步骤 2: 数据处理

- **代理**: data-processor
- **输入**:
  - data: 来自步骤 1
- **输出**:
  - processed_data: 处理后的数据
- **条件**: data 不为空

### 步骤 3: 生成报告

- **代理**: report-generator
- **输入**:
  - processed_data: 来自步骤 2
- **输出**:
  - report: 最终报告
```

### 步骤 4: 定义条件分支

```markdown
## 条件分支

### 分支 A: 成功路径

- 条件: result.status == 'success'
- 下一步: 步骤 4

### 分支 B: 需要重试

- 条件: result.status == 'retry'
- 下一步: 步骤 1

### 分支 C: 失败

- 条件: result.status == 'error'
- 下一步: 错误处理
```

### 步骤 5: 异常处理

```markdown
## 异常处理

### 超时处理

- 动作: 记录日志并通知用户
- 恢复: 提供部分结果

### 错误处理

- 动作: 回滚已完成的步骤
- 恢复: 提供错误报告
```

```markdown

## 概述

[工作流描述]

## 触发条件

- [触发条件 1]
- [触发条件 2]

## 前置条件

- [前置条件 1]
- [前置条件 2]

## 步骤

### 步骤 1: [步骤名称]

- **代理**: [代理名称]
- **描述**: [步骤描述]
- **输入**:
  - [输入项]: [描述]
- **输出**:
  - [输出项]: [描述]

### 步骤 2: [步骤名称]

...

## 输出

- [最终输出描述]

## 异常处理

- [异常场景及处理方式]
```

### 并行执行模板

```markdown
## 并行步骤

### 并行组 1

以下步骤并行执行：

#### 步骤 1A: [名称]

- **代理**: [代理]
- **输入**: ...

#### 步骤 1B: [名称]

- **代理**: [代理]
- **输入**: ...

### 同步点

等待并行组 1 完成后，继续执行步骤 2
```

## 💡 最佳实践

### 1. 保持步骤原子化

每个步骤应该只做一件事：

```markdown
# 好的做法

步骤 1: 收集数据
步骤 2: 验证数据
步骤 3: 处理数据

# 不好的做法

步骤 1: 收集、验证并处理数据
```

### 2. 明确定义输入输出

```markdown
# 好的做法

- **输入**:
  - user_id: string - 用户唯一标识
  - date_range: object - 日期范围
    - start: date
    - end: date

# 不好的做法

- **输入**: 用户信息和日期
```

### 3. 处理边界情况

```markdown
## 边界情况

### 空输入

- 检测: input == null
- 处理: 返回错误提示

### 超大数据

- 检测: data.size > limit
- 处理: 分批处理
```

### 4. 添加超时和重试

```markdown
### 步骤 1: 外部 API 调用

- **超时**: 30秒
- **重试**: 3次
- **重试间隔**: 指数退避 (1s, 2s, 4s)
```

## 🔧 调试工作流

### 日志记录

```markdown
## 日志配置

- 级别: INFO
- 输出:
  - 步骤开始/结束
  - 输入/输出摘要
  - 错误详情
```

### 测试模式

```bash
# 运行工作流测试
bmad workflow test my-workflow

# 调试模式
bmad workflow test my-workflow --debug
```
