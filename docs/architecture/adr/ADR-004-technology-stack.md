# ADR-004: 技术栈选择

## 状态

✅ 已采纳

## 上下文

cloudQuant 量化交易平台需要选择合适的技术栈来支持：

- 高性能策略回测
- 实时行情处理
- 多交易所接入
- 数据分析和可视化
- AI/ML 功能集成

## 决策

### 核心语言

| 语言           | 用途               | 理由                       |
| -------------- | ------------------ | -------------------------- |
| **Python**     | 策略开发、数据分析 | 丰富的量化库生态、快速开发 |
| **TypeScript** | Web 前端、API 服务 | 类型安全、良好的开发体验   |
| **Rust**       | 高性能计算模块     | 零成本抽象、内存安全       |

### 数据存储

| 存储            | 用途           | 理由                |
| --------------- | -------------- | ------------------- |
| **PostgreSQL**  | 结构化数据     | ACID 保证、复杂查询 |
| **TimescaleDB** | 时序数据       | 高效的时间序列存储  |
| **Redis**       | 缓存、实时数据 | 高性能、低延迟      |

### 消息队列

| 技术              | 用途       | 理由           |
| ----------------- | ---------- | -------------- |
| **Kafka**         | 行情数据流 | 高吞吐、持久化 |
| **Redis Streams** | 内部事件   | 简单、低延迟   |

### AI/ML 框架

| 框架         | 用途         |
| ------------ | ------------ |
| PyTorch      | 深度学习模型 |
| scikit-learn | 传统机器学习 |
| Ray          | 分布式计算   |

### 前端技术

| 技术         | 用途     |
| ------------ | -------- |
| React        | UI 框架  |
| TypeScript   | 类型安全 |
| Tailwind CSS | 样式系统 |
| Vite         | 构建工具 |

## 后果

### 正面影响

- Python 生态丰富，降低开发成本
- TypeScript 提供良好的类型安全
- Rust 关键路径性能优化
- 技术栈成熟，社区支持好

### 负面影响

- 多语言增加运维复杂度
- 团队需要掌握多种技术
- 跨语言调用需要额外设计

### 风险缓解

- 使用标准化接口（REST/gRPC）
- 完善的文档和培训
- 逐步引入，避免大爆炸重构
