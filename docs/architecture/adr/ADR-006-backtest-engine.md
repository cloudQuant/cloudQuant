# ADR-006: 回测引擎架构

## 状态

✅ 已采纳

## 上下文

回测引擎是量化交易平台的核心组件，需要：

- 高性能：支持快速迭代
- 准确性：模拟真实交易环境
- 灵活性：支持多种策略类型
- 可扩展：分布式回测能力

## 决策

### 核心组件

| 组件                  | 职责         | 技术实现       |
| --------------------- | ------------ | -------------- |
| DataLoader            | 加载历史数据 | Pandas + Numba |
| EventEngine           | 事件调度     | 事件循环       |
| StrategyRunner        | 策略执行     | Python 解释器  |
| OrderMatcher          | 订单撮合     | 滑点模型       |
| RiskChecker           | 风控检查     | 规则引擎       |
| PerformanceCalculator | 绩效计算     | Numba 加速     |

### 事件驱动模型

```python
class Event:
    type: EventType  # BAR, TICK, ORDER, TRADE
    timestamp: datetime
    data: dict

class EventEngine:
    def on_bar(self, bar: Bar):
        self.strategy.on_bar(bar)
        self.match_pending_orders()
        self.check_risk()
```

### 性能优化策略

1. **数据预加载**
   - 使用内存映射文件
   - Parquet 列式存储
   - 增量加载机制

2. **计算加速**
   - Numba JIT 编译
   - 向量化计算
   - 多进程并行

3. **缓存策略**
   - 策略指标缓存
   - 中间结果缓存

### 回测模式

| 模式      | 描述        | 适用场景 |
| --------- | ----------- | -------- |
| 向量模式  | 批量计算    | 快速验证 |
| 事件模式  | 逐 bar 模拟 | 精确回测 |
| Tick 模式 | 逐笔模拟    | 高频策略 |

## 后果

### 正面影响

- 事件驱动模型准确模拟真实交易
- Numba 加速带来 10-100x 性能提升
- 分布式回测支持大规模参数优化
- 模块化设计便于扩展

### 负面影响

- 事件驱动比向量模式慢
- 分布式增加系统复杂度
- 需要更多内存资源

### 替代方案

| 方案       | 为何不选         |
| ---------- | ---------------- |
| 纯向量回测 | 无法模拟日内交易 |
| 基于数据库 | 性能瓶颈         |
| 第三方引擎 | 定制性差         |
