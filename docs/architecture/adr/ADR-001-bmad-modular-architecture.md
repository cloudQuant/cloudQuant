# ADR-001: BMAD 模块化架构

| 状态 | 日期 | 决策者 |
|------|------|--------|
| 已采纳 | 2026-03-13 | cloudQuant |

## 上下文

cloudQuant 项目需要一个可扩展的 AI 辅助开发框架，支持多种开发场景：

- 量化交易策略开发
- 游戏开发
- Web 应用开发
- 测试工程

需要一种模块化架构，使得不同领域的开发团队能够独立使用所需的工具集。

## 决策

采用 BMAD (Breakthrough Method for AI Development) 框架的模块化架构设计:

1. **核心模块 (Core)**: 提供基础工作流和代理系统
2. **业务方法论模块 (BMM)**: 敏捷开发、PRD、架构设计
3. **创意智能套件 (CIS)**: 设计思维、头脑风暴、创新策略
4. **游戏开发套件 (GDS)**: 游戏架构、开发流程、测试
5. **测试工程架构 (TEA)**: 测试框架、自动化测试、CI 集成
6. **Web 设计系统 (WDS)**: UI 设计、组件库、工作流

## 后果

### 正面影响

- ✅ 各模块独立维护和更新
- ✅ 支持多种 IDE (Claude Code, Cursor, Codex, OpenCode, Windsurf)
- ✅ 模块可按需安装，减少不必要的配置
- ✅ 清晰的关注点分离

### 负面影响

- ⚠️ 配置文件分散在多个目录
- ⚠️ 需要额外的文档说明各模块关系
- ⚠️ 新用户可能需要时间理解整体架构

### 替代方案

1. **单体配置**: 所有配置放在一个文件中
   - 优点: 简单
   - 缺点: 难以维护和扩展

2. **完全分离的仓库**: 每个模块独立仓库
   - 优点: 完全解耦
   - 缺点: 增加管理复杂度
